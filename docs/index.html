Project Management AI Exposure Index

This dashboard measures the extent to which agents can do project-management tasks as defined by O*NET. It combines a fixed occupation-level baseline with current public benchmark evidence so you can compare how capable different systems appear to be against the same PM task bundle.

Higher scores mean stronger evidence that an agent can carry out more of the O*NET project-management task set under the benchmarks currently available in the dashboard.

Structural Baseline

The structural baseline is the occupation side of the index. It does not rate any individual model. Instead, it summarizes the O*NET project-management task bundle itself: how much of the work appears technically exposable to agents, how tightly the tasks connect to each other, and how much human oversight still matters. The comparison table below changes by model and benchmark evidence, but these baseline values stay fixed unless the underlying O*NET task coding changes.

Direct Exposure

The share of the O*NET PM task bundle that looks performable, accelerable, or partially substitutable by agents when you examine the task content directly.

Spillover Exposure

The extra impact created when a task sits in the middle of other tasks. If agents can do a highly connected task, the effect can spill across planning, reporting, monitoring, and change decisions.

Oversight Criticality

The degree to which the work still depends on human approval, accountability, escalation, stakeholder judgment, or responsibility even when agents can help with execution.

Interdependence

How tightly the PM tasks influence each other. Higher interdependence means the occupation is less like a list of isolated tasks and more like a connected operating system.

Task Count

The number of O*NET PM tasks included in the current pilot occupation. This keeps the scope of the baseline transparent.

Rank

The position of the system after sorting the current table view. By default, rows are ordered by realized exposure from highest to lowest.

System

The tracked model or agent entry being compared. This is the unit as it appears in the benchmark pipeline, which can be a base model, a tool-using system, or an agent scaffold.

Provider

The organization, product family, or benchmark submitter associated with the system label. This helps you group related entries without assuming they are identical systems.

Realized Exposure

The main headline score. It estimates how far the system appears able to perform the O*NET PM task bundle once benchmark evidence is layered on top of the structural baseline.

Coverage

The share of the capability layer supported by benchmark evidence for that system in the current snapshot. Lower coverage means a row is supported by less benchmark visibility.

Benchmarks

The benchmark families currently contributing evidence for that system. These tags show where the evidence came from rather than acting as a quality score by themselves.

How To Read This

Higher realized exposure means the system aligns more strongly with the O*NET PM task structure under the current benchmark evidence. It is not a claim of full automation or job substitution.

What Coverage Means

Coverage is the weighted share of the capability layer supported by benchmark results for a given system in the current snapshot. Low coverage means interpret rankings carefully.

Monthly Update Path

Refresh benchmark results, rebuild the monthly snapshot, then rerun the site-data builder. The table is designed to update without changing the front-end code.
