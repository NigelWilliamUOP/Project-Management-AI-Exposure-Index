This version compares underlying frontier LLM model families rather than benchmark-specific agent systems. It measures the extent to which those model families appear able to do project-management tasks as defined by O*NET.

For each benchmark in the current snapshot, the dashboard keeps the best observed public system entry using a given model family, then recomputes the PM exposure scores from those family-level benchmark rows.

How This View Works

The structural baseline is still the same O*NET project-management occupation. What changes here is the capability layer: instead of ranking benchmark-specific systems, this page collapses public benchmark rows into frontier model families and then recomputes the PM exposure scores.

Aggregation Rule

For each model family and benchmark, the view keeps the highest-scoring matched public system entry in the current snapshot.

What It Is Good For

This is useful when you want to compare underlying model families rather than specific products, wrappers, or agent scaffolds.

What It Is Not

This is not a scaffold-neutral lab test. It is a best-observed public-system view of each frontier family under the benchmarks currently available.

Current Coverage Limit

In the current April 2026 snapshot, most frontier model families are observed primarily through Terminal-Bench 2.0, so coverage is still thin and should be read as directional rather than final.
